# 🚀 Быстрый старт Imperion-Pro

Краткое руководство для запуска проекта за 5 минут.

## 📋 Что это?

**Imperion-Pro** — интерактивная платформа для изучения программирования с практическими уроками по Python, JavaScript, C++, C и Go.

## ⚡ Установка (2 минуты)

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/Imperion-Pro.git
cd Imperion-Pro
```

### 2. Установите зависимости

```bash
npm install
```

### 3. Настройте переменные окружения

Создайте файл `.env.local`:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

> 📖 **Где взять ключи?** Создайте проект на [supabase.com](https://supabase.com) и скопируйте URL и anon key из Settings → API

## 🗄️ База данных (2 минуты)

### 1. Примените схему

В Supabase SQL Editor выполните содержимое файла `supabase/schema.sql`

### 2. Загрузите данные

В Supabase SQL Editor выполните содержимое файла `supabase/seed.sql`

> 📖 **Подробная инструкция:** См. `DATABASE_SETUP.md`

## 🎯 Запуск (1 минута)

```bash
npm run dev
```

Откройте [http://localhost:5173](http://localhost:5173)

## 👤 Первый вход

1. Зарегистрируйтесь через интерфейс
2. Подтвердите email (или используйте Supabase для подтверждения)
3. Готово! 🎉

## 🔐 Назначение ролей

По умолчанию все пользователи имеют роль **"пользователь"**.

### Сделать себя администратором

В Supabase SQL Editor:

```sql
UPDATE public.profiles 
SET role = 'администратор' 
WHERE id = 'YOUR_USER_UUID';
```

> 💡 UUID можно найти в Authentication → Users

## 🆘 Частые проблемы

### Ошибка подключения к БД
- Проверьте правильность `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`
- Убедитесь, что проект Supabase активен

### Не отображаются уроки
- Убедитесь, что выполнили `seed.sql`
- Проверьте, что уроки опубликованы (`published = true`)

### "Permission denied" при создании урока
- Убедитесь, что у вашего пользователя роль `ментор` или `администратор`
- Проверьте, что схема БД применена корректно

### Статусы на английском
- Выполните миграцию из `CHANGELOG.md`
- Переприменитие `schema.sql`
