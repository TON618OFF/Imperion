# Настройка базы данных Imperion

Данная инструкция поможет вам настроить базу данных PostgreSQL через Supabase для проекта Imperion.

## Шаг 1: Создание проекта в Supabase

1. Перейдите на [https://supabase.com](https://supabase.com)
2. Зарегистрируйтесь или войдите в аккаунт
3. Нажмите "New Project"
4. Заполните данные проекта:
   - Название проекта: `imperion`
   - Database Password: создайте надёжный пароль (сохраните его!)
   - Region: выберите ближайший регион
5. Нажмите "Create new project" и дождитесь завершения создания (1-2 минуты)

## Шаг 2: Получение учётных данных

После создания проекта:

1. Перейдите в раздел **Settings** → **API**
2. Скопируйте следующие данные:
   - **Project URL** (например: `https://xxxxx.supabase.co`)
   - **anon public key** (публичный ключ для клиентских запросов)

## Шаг 3: Настройка переменных окружения

Создайте файл `.env.local` в корне проекта:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

⚠️ **Важно:** Не коммитьте файл `.env.local` в репозиторий! Он уже добавлен в `.gitignore`.

## Шаг 4: Применение базы данных

1. В Supabase перейдите в раздел **SQL Editor**
2. Нажмите "New query"
3. Откройте файл `supabase/database_complete.sql` из проекта
4. Скопируйте **всё содержимое** файла и вставьте в SQL Editor
5. Нажмите "Run" (или Ctrl+Enter)
6. Дождитесь успешного выполнения (10-30 секунд, должно появиться сообщение "Success")

**Этот файл содержит всё необходимое:**
- Все таблицы (profiles, lessons, lectures, achievements и др.)
- Enum типы для ролей, статусов, языков и уровней сложности
- Функции для работы с XP, достижениями и прогрессом
- Row Level Security политики
- Триггеры для автоматического обновления данных
- 45 уроков по всем 5 языкам программирования (9 на каждый язык)
- 14 лекций с теоретическим материалом
- 15 достижений
- Подсказки к урокам

## Шаг 5: Создание тестовых пользователей

### 5.1 Регистрация через приложение

1. Запустите приложение: `npm run dev`
2. Перейдите на страницу регистрации
3. Зарегистрируйте пользователя
4. Подтвердите email (проверьте почту или используйте Supabase для подтверждения)

### 5.2 Назначение ролей пользователям

По умолчанию все пользователи имеют роль **"пользователь"**. Чтобы назначить роли **"ментор"** или **"администратор"**:

1. Перейдите в **Authentication** → **Users** в Supabase
2. Скопируйте UUID нужного пользователя
3. Перейдите в **SQL Editor** и выполните запрос:

```sql
-- Назначить роль "администратор"
UPDATE public.profiles 
SET role = 'администратор' 
WHERE id = 'USER_UUID_HERE';

-- Назначить роль "ментор"
UPDATE public.profiles 
SET role = 'ментор' 
WHERE id = 'USER_UUID_HERE';

-- Назначить роль "пользователь" (по умолчанию)
UPDATE public.profiles 
SET role = 'пользователь' 
WHERE id = 'USER_UUID_HERE';
```

Замените `USER_UUID_HERE` на реальный UUID пользователя.

## Шаг 6: Настройка Email Authentication

Для работы аутентификации:

1. Перейдите в **Authentication** → **Providers**
2. Убедитесь, что **Email** включён
3. В разделе **Email Templates** настройте шаблоны писем (опционально)
4. В **URL Configuration** установите:
   - **Site URL**: `http://localhost:8080` (для разработки)
   - **Redirect URLs**: добавьте `http://localhost:8080/**`

## Шаг 7: Проверка работоспособности

### Проверка таблиц

Выполните в SQL Editor:

```sql
-- Проверить количество уроков
SELECT language, difficulty, COUNT(*) 
FROM public.lessons 
GROUP BY language, difficulty 
ORDER BY language, difficulty;

-- Проверить количество лекций
SELECT language, COUNT(*) 
FROM public.lectures 
GROUP BY language 
ORDER BY language;

-- Проверить достижения
SELECT title, rarity 
FROM public.achievements 
WHERE published = true;
```

### Проверка ролей пользователей

```sql
SELECT 
  p.id,
  p.username,
  p.first_name,
  p.last_name,
  p.role,
  au.email
FROM public.profiles p
JOIN auth.users au ON au.id = p.id;
```

## Описание ролей

### Роль "пользователь" (по умолчанию)
- Доступ ко всем урокам и лекциям
- Отслеживание прогресса
- Получение достижений
- Редактирование своего профиля

### Роль "ментор"
- Все права пользователя
- Доступ к **Панели ментора** (`/mentor`)
- Создание, редактирование и удаление уроков
- Создание, редактирование и удаление лекций
- Управление контентом для всех языков
- **НЕ имеет доступа** к административной панели

### Роль "администратор"
- Все права ментора
- Доступ к **Панели администратора** (`/admin`)
- Просмотр полной статистики платформы
- Управление пользователями
- Доступ к аналитике и графикам
- Экспорт данных в PDF

## Часто возникающие проблемы

### Ошибка "new row violates row-level security policy"

**Причина:** RLS политики блокируют операцию.

**Решение:** Убедитесь, что пользователь аутентифицирован и имеет нужную роль.

### Ошибка при регистрации пользователя

**Причина:** Триггер `on_auth_user_created` не создал запись в `profiles`.

**Решение:** Выполните функцию вручную:

```sql
SELECT public.bootstrap_current_user();
```

### Не отображаются уроки для ментора

**Причина:** Возможно не применены политики безопасности для ментора.

**Решение:** Повторно примените скрипт `database_complete.sql`.

### Статусы отображаются на английском

**Причина:** Старые данные в базе.

**Решение:** Обновите статусы:

```sql
UPDATE public.user_lesson_progress 
SET status = CASE 
  WHEN status = 'not_started' THEN 'не_начат'
  WHEN status = 'in_progress' THEN 'в_процессе'
  WHEN status = 'completed' THEN 'завершён'
  ELSE status
END;
```

## Storage для аватаров

Bucket `avatars` уже создан схемой и настроен как публичный. Политики RLS настроены так, что:
- Любой аутентифицированный пользователь может читать аватары
- Пользователь может загружать/обновлять только свой аватар

Следите за использованием:
1. **Reports** — статистика использования
2. **Database** → **Logs** — логи запросов
