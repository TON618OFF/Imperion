## О роли ментора

### Ваши возможности

Как ментор вы можете:
- ✅ Создавать, редактировать и удалять **уроки**
- ✅ Настраивать **контент уроков**: задания, код, проверки
- ✅ Добавлять **подсказки** к урокам
- ✅ Создавать, редактировать и удалять **лекции**
- ✅ Управлять **секциями лекций** с теоретическим материалом
- ✅ Работать со всеми **5 языками программирования**

### Чего вы НЕ можете делать

- ❌ Управлять пользователями
- ❌ Просматривать полную статистику (доступно только администраторам)
- ❌ Назначать роли другим пользователям

## Создание уроков

### Шаг 1: Создание базового урока

1. Перейдите на вкладку **"Уроки"**
2. Нажмите кнопку **"Создать урок"**
3. Заполните форму:

#### Обязательные поля:

**Slug (URL)**
- Уникальный идентификатор урока для URL
- Используйте kebab-case: `python-hello-world`
- Только латинские буквы, цифры и дефисы
- Примеры: `py-strings`, `js-async`, `cpp-pointers`

**Название**
- Понятное название на русском языке
- Включите язык программирования: `Python: Hello World`
- Будет отображаться в списке уроков

**Язык программирования**
- Python
- JavaScript
- C++
- C
- Go

**Уровень сложности**
- 🟢 **Лёгкий** — для начинающих (базовые концепции)
- 🟡 **Средний** — требует базовых знаний
- 🔴 **Сложный** — продвинутые темы

**Тип урока**
- **Практика** — задание с проверкой кода (рекомендуется)
- **Теория** — без проверки кода
- **Экзамен** — итоговое задание

#### Опциональные поля:

**Описание**
- Краткое описание урока (1-2 предложения)
- Отображается в карточке урока

**Расчётное время (минуты)**
- Сколько времени займет урок
- Помогает ученикам планировать обучение

**Порядковый номер**
- Определяет позицию в списке
- Чем меньше число, тем раньше урок

**Статус публикации**
- ☑️ **Опубликовать** — урок виден всем
- ☐ **Черновик** — виден только менторам

4. Нажмите **"Сохранить"**

⚠️ **Важно:** На этом этапе урок создан, но у него ещё нет контента!

## Настройка контента урока

### Шаг 2: Добавление контента

После создания урока нужно настроить его содержимое:

1. В таблице уроков найдите свой урок
2. Нажмите кнопку с иконкой **📄** (Управление контентом)
3. Откроется диалог настройки контента

### Описание задания (Markdown)

Напишите задание для ученика в формате Markdown:

```markdown
# Урок 1: Hello World

В этом задании вы напишете свою первую программу.

## Задача

Выведите на экран текст `Hello, world!`

## Требования

- Используйте функцию `print()` для Python
- Вывод должен быть точно `Hello, world!` (без лишних символов)
- Программа не должна выдавать ошибок

## Подсказка

В Python для вывода текста используется функция:
```python
print("текст")
```
```

**Поддерживаемый Markdown:**
- Заголовки: `#`, `##`, `###`
- Жирный текст: `**текст**`
- Курсив: `*текст*`
- Блоки кода: ` ```python ` ... ` ``` `
- Списки: `- пункт` или `1. пункт`
- Ссылки: `[текст](url)`

### Стартовый код

Код, который ученик увидит при открытии урока:

**Для Python:**
```python
# Напишите ваш код здесь
print("Hello, world!")
```

**Для JavaScript:**
```javascript
// Напишите ваш код здесь
console.log("Hello, world!");
```

**Для C++:**
```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, world!" << endl;
    return 0;
}
```

**Для C:**
```c
#include <stdio.h>

int main() {
    printf("Hello, world!\n");
    return 0;
}
```

**Для Go:**
```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, world!")
}
```

💡 **Совет:** Можно оставить пустым или дать шаблон с комментариями.

### Ожидаемый результат (expected_stdout)

**ЭТО САМОЕ ВАЖНОЕ ПОЛЕ!**

Введите точный результат, который должна вывести программа:

```
Hello, world!
```

**Как это работает:**

1. Ученик пишет код и нажимает "Запустить"
2. Код отправляется в **Piston API** для выполнения
3. Piston возвращает вывод программы (stdout)
4. Система сравнивает вывод с вашим `expected_stdout`
5. Если совпадает → ✅ урок можно завершить
6. Если не совпадает → ❌ нужно исправить код

**Особенности проверки:**

- ✅ Игнорируются конечные пробелы
- ✅ Игнорируются конечные переводы строк
- ✅ `\r\n` и `\n` считаются одинаковыми
- ❌ Регистр букв ВАЖЕН: `hello` ≠ `Hello`
- ❌ Порядок строк ВАЖЕН

**Примеры:**

**Простой вывод:**
```
Hello, world!
```

**Несколько строк:**
```
1
2
3
4
5
```

**Форматированный вывод:**
```
Имя: Иван
Возраст: 25
Город: Москва
```

**С пробелами:**
```
x = 10
y = 20
сумма = 30
```

⚠️ **Внимание:** Если не задать `expected_stdout`, урок нельзя будет завершить!

### Подсказки

Добавьте подсказки, которые помогут ученикам:

1. Нажмите **"Добавить подсказку"**
2. Введите текст подсказки
3. Можно добавить несколько подсказок
4. Подсказки показываются по порядку

**Примеры хороших подсказок:**

**Подсказка 1 (общая):**
```
Используйте функцию print() для вывода текста
```

**Подсказка 2 (конкретная):**
```
Попробуйте: print("Hello, world!")
```

**Подсказка 3 (с кодом):**
```
Полное решение:

print("Hello, world!")

Обратите внимание на кавычки и скобки!
```

💡 **Совет:** Не давайте полное решение в первой подсказке!

### Шаг 3: Сохранение контента

Нажмите **"Сохранить контент"**

✅ Теперь урок полностью готов для учеников!

## Создание лекций

Лекции — это теоретический материал без проверки кода.

### Шаг 1: Создание базовой лекции

1. Перейдите на вкладку **"Лекции"**
2. Нажмите **"Создать лекцию"**
3. Заполните форму:

**Slug (URL)**
- Уникальный идентификатор: `python-basics`

**Название**
- `Python: Основы программирования`

**Язык программирования**
- Выберите из списка

**Краткое описание**
- Что изучит ученик в этой лекции
- Пример: `Переменные, типы данных, операторы`

**Порядковый номер**
- Позиция в списке лекций

**Статус публикации**
- Опубликовать или оставить черновиком

4. Нажмите **"Сохранить"**

### Шаг 2: Добавление секций

Лекция состоит из секций — это отдельные части материала:

1. В таблице лекций нажмите кнопку **📄** (Управление секциями)
2. Нажмите **"Добавить секцию"**
3. Для каждой секции укажите:

**Заголовок секции**
- Название раздела: `Введение`, `Переменные`, `Типы данных`

**Контент (Markdown)**
- Полный текст секции в Markdown

**Пример секции:**

```markdown
# Введение в Python

Python — это высокоуровневый язык программирования.

## Особенности Python

- Простой синтаксис
- Интерпретируемый язык
- Большое сообщество
- Множество библиотек

## Первая программа

Самая простая программа на Python:

```python
print("Hello, world!")
```

Эта программа выводит текст на экран.

## Что дальше?

В следующих секциях мы изучим:
- Переменные
- Типы данных
- Операторы
```

4. Добавьте все необходимые секции
5. Нажмите **"Сохранить секции"**

✅ Лекция готова!

## Best Practices

### Для уроков

#### ✅ Хорошо:
- Одна концепция = один урок
- Чёткая формулировка задания
- Примеры в описании
- Стартовый код с комментариями
- Несколько уровней подсказок
- Точный expected_stdout

#### ❌ Плохо:
- Слишком сложное задание для начинающих
- Нечёткие требования
- Отсутствие подсказок
- Неправильный expected_stdout
- Опечатки в описании

### Для лекций

#### ✅ Хорошо:
- Логическая структура секций
- Примеры кода в каждой секции
- Визуальное форматирование (списки, заголовки)
- Связь с практическими уроками
- Постепенное усложнение

#### ❌ Плохо:
- Слишком длинные секции (разбейте!)
- Текст без форматирования
- Отсутствие примеров кода
- Слишком сложные термины без объяснений

### Общие рекомендации

1. **Тестируйте уроки сами**
   - Попробуйте выполнить задание
   - Проверьте, что expected_stdout правильный
   - Убедитесь, что подсказки помогают

2. **Следуйте прогрессии сложности**
   - Лёгкий → Средний → Сложный
   - От простого к сложному внутри уровня

3. **Используйте примеры из жизни**
   - Практические задачи интереснее
   - Связь с реальными приложениями

4. **Проверяйте орфографию**
   - Опечатки снижают доверие
   - Используйте проверку правописания

# Что такое указатель?

Указатель — это переменная, которая хранит адрес другой переменной.

## ❓ Частые вопросы

### Как исправить урок, если нашел ошибку?

1. Найдите урок в таблице
2. Нажмите кнопку "Редактировать" (карандаш)
3. Исправьте метаданные
4. Для исправления контента: кнопка 📄 → исправьте → Сохранить

### Могу ли я удалить опубликованный урок?

Да, но **будьте осторожны**:
- Удаление необратимо
- Прогресс учеников по этому уроку будет потерян
- Лучше сначала снять публикацию (сделать черновиком)

### Что если expected_stdout неправильный?

Ученики не смогут завершить урок! Обязательно:
1. Проверьте вывод сами в консоли
2. Скопируйте ТОЧНЫЙ вывод (включая пробелы)
3. Протестируйте урок под тестовым аккаунтом

### Можно ли использовать HTML в Markdown?

Нет, используйте только стандартный Markdown:
- Заголовки, списки, код
- Жирный, курсив
- Ссылки

### Сколько подсказок оптимально?

Рекомендуется **2-3 подсказки**:
1. Общая подсказка (направление)
2. Конкретная подсказка (как сделать)
3. Решение (если совсем сложно)
