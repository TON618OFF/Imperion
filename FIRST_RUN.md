# 🎯 Первый запуск Imperion

Пошаговая инструкция для первого запуска проекта. Следуйте шагам по порядку.

---

## ✅ Чек-лист перед началом

Убедитесь, что у вас установлено:
- [ ] Node.js (версия 16 или выше) - `node --version`
- [ ] npm (обычно идёт с Node.js) - `npm --version`
- [ ] Git - `git --version`
- [ ] Аккаунт на [supabase.com](https://supabase.com)
- [ ] Текстовый редактор (VSCode, WebStorm и т.д.)

---

## 📦 Шаг 1: Клонирование и установка (3 минуты)

### 1.1 Клонируйте репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/Imperion.git
cd Imperion
```

### 1.2 Установите зависимости

```bash
npm install
```

⏱️ Это займёт 2-3 минуты. Дождитесь завершения установки.

### 1.3 Проверка установки

```bash
npm run lint
```

✅ Если команда выполнилась без ошибок - всё хорошо!

---

## 🗄️ Шаг 2: Настройка Supabase (5 минут)

### 2.1 Создайте проект в Supabase

1. Откройте [https://supabase.com](https://supabase.com)
2. Войдите в аккаунт или зарегистрируйтесь
3. Нажмите **"New Project"**
4. Заполните форму:
   - **Name**: `imperion-pro`
   - **Database Password**: придумайте надёжный пароль (сохраните его!)
   - **Region**: выберите ближайший к вам
   - **Pricing Plan**: `Free` (для разработки)
5. Нажмите **"Create new project"**
6. ⏱️ Подождите 1-2 минуты пока проект создаётся

### 2.2 Получите ключи доступа

1. После создания проекта перейдите в **Settings** (⚙️ в левом меню)
2. Выберите **API**
3. Скопируйте два значения:
   - **Project URL** (например: `https://abcdefgh.supabase.co`)
   - **anon public** key (длинная строка)

### 2.3 Создайте файл с переменными окружения

В корне проекта создайте файл `.env.local`:

```env
VITE_SUPABASE_URL=вставьте_сюда_project_url
VITE_SUPABASE_ANON_KEY=вставьте_сюда_anon_key
```

⚠️ **ВАЖНО:**
- Замените текст после `=` на ваши реальные значения
- Не используйте кавычки
- Не добавляйте пробелы вокруг `=`

**Пример:**
```env
VITE_SUPABASE_URL=https://abcdefgh.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 🎨 Шаг 3: Настройка базы данных (5 минут)

### 3.1 Откройте SQL Editor в Supabase

1. В панели Supabase перейдите в **SQL Editor** (иконка 📝)
2. Нажмите **"New query"**

### 3.2 Примените схему базы данных

1. Откройте файл `supabase/schema.sql` в вашем редакторе
2. Скопируйте **ВСЁ** содержимое файла (Ctrl+A, Ctrl+C)
3. Вставьте в SQL Editor Supabase (Ctrl+V)
4. Нажмите **"Run"** (или нажмите Ctrl+Enter)
5. ⏱️ Подождите 10-15 секунд

✅ В правом нижнем углу должно появиться сообщение **"Success. No rows returned"**

❌ Если появилась ошибка:
- Проверьте, что скопировали весь файл
- Убедитесь, что проект Supabase полностью загрузился
- Попробуйте ещё раз

### 3.3 Загрузите начальные данные

1. Нажмите **"New query"** для нового запроса
2. Откройте файл `supabase/seed.sql`
3. Скопируйте **ВСЁ** содержимое
4. Вставьте в новый SQL Editor
5. Нажмите **"Run"**
6. ⏱️ Подождите 5-10 секунд

✅ Должно появиться **"Success. No rows returned"**

### 3.4 Проверка данных

Выполните в SQL Editor:

```sql
-- Проверить уроки
SELECT COUNT(*) as total_lessons FROM public.lessons;

-- Проверить лекции  
SELECT COUNT(*) as total_lectures FROM public.lectures;

-- Проверить достижения
SELECT COUNT(*) as total_achievements FROM public.achievements;
```

✅ Результаты должны быть:
- `total_lessons`: 15
- `total_lectures`: 5
- `total_achievements`: 10

## 👤 Шаг 5: Создание первого пользователя (3 минуты)

### 5.1 Регистрация

1. На главной странице нажмите **"Начать обучение"** или **"Войти"**
2. Перейдите на вкладку **"Регистрация"**
3. Заполните форму:
   - Email (используйте реальный email)
   - Пароль (минимум 6 символов)
4. Нажмите **"Зарегистрироваться"**

### 5.2 Подтверждение email

**Вариант А: Через почту** (если настроен SMTP)
- Проверьте входящие письма
- Кликните на ссылку подтверждения

**Вариант Б: Через Supabase** (если письмо не пришло)
1. В Supabase перейдите в **Authentication** → **Users**
2. Найдите вашего пользователя
3. Кликните на три точки (...) справа
4. Выберите **"Send confirmation email"**
5. ИЛИ кликните **"Confirm user"** для мгновенного подтверждения

### 5.3 Вход в систему

1. После подтверждения вернитесь на страницу входа
2. Введите email и пароль
3. Нажмите **"Войти"**

✅ Вы должны попасть в личный кабинет (Dashboard)

### 6.1 Проверьте уроки

1. Перейдите в раздел **"Уроки"**

### 6.2 Проверьте лекции

2. Выберите язык
3. Откройте лекцию
4. Прочитайте содержимое

✅ Лекция должна отображаться с форматированием Markdown

### 6.3 Проверьте профиль

1. Перейдите в **"Профиль"**
2. Заполните имя и фамилию
3. Сохраните

✅ Данные должны сохраниться

## 👨‍💼 Шаг 7: Назначение ролей (опционально)

### 7.1 Получите ваш User ID

1. В Supabase перейдите в **Authentication** → **Users**
2. Найдите вашего пользователя
3. Скопируйте **UUID** (длинная строка с дефисами)

### 7.2 Сделайте себя администратором

В SQL Editor выполните:

```sql
UPDATE public.profiles 
SET role = 'администратор' 
WHERE id = 'ВСТАВЬТЕ_СЮДА_ВАШ_UUID';
```

**Пример:**
```sql
UPDATE public.profiles 
SET role = 'администратор' 
WHERE id = '12345678-1234-1234-1234-123456789abc';
```

✅ Должно появиться: **"Success. 1 rows affected"**

### 7.3 Проверьте доступ к панелям

1. Обновите страницу приложения (F5)
2. В меню должны появиться:
   - **"Панель ментора"**
   - **"Админ-панель"**
3. Перейдите в каждую панель

✅ Панели должны открыться без ошибок

## 🆘 Возникли проблемы?

### Ошибка "Failed to fetch"
**Решение:** Проверьте `.env.local` - правильные ли там URL и ключ

### Ошибка "Invalid API key"
**Решение:** Скопируйте anon key заново из Supabase

### Уроки не отображаются
**Решение:** Проверьте, что выполнили `seed.sql`

### "Permission denied" при создании урока
**Решение:** Назначьте роль "ментор" или "администратор"

### Код не выполняется
**Решение:** Проверьте подключение к интернету (используется Piston API)

### Другая проблема?
1. Проверьте консоль браузера (F12)
2. Проверьте логи в терминале
3. Прочитайте `DATABASE_SETUP.md`
4. Создайте issue на GitHub
